# team-tasks 페르소나

사내 일감 할당 앱의 핵심 사용자 3명. MVP 헌장(단일 테이블 · 기능 5개 이하 · 1일 구현)을 만족하도록 시나리오는 "일감 1건의 생성 → 할당 → 완료" 흐름 안에서만 정의한다.

| 역할 | 기술 수준 | 주요 시나리오 3개 |
| --- | --- | --- |
| **팀 리드 (지수, 백엔드 팀장)** | 사내 툴(Jira, Notion) 능숙 · CLI 가능 · 일 단위로 일감 분배 | 1) 새 일감을 제목·담당자만 지정해 즉시 등록한다. 2) 팀원별 미완료 일감 수를 한눈에 보고 과부하인 사람에게서 다른 사람으로 재할당한다. 3) 완료된 일감을 주간 회고 전에 모아 본다. |
| **실무 담당자 (민호, 3년 차 개발자)** | 웹앱 사용은 능숙하나 관리 기능엔 무관심 · 모바일에서도 본다 | 1) "내 일감"만 필터해 오늘 할 일을 확인한다. 2) 진행 중인 일감을 "완료"로 한 번에 토글한다. 3) 내용이 모호한 일감에 한 줄 메모(설명 필드)를 덧붙여 리드에게 의도를 다시 묻지 않게 한다. |
| **신규 입사자 (서연, 인턴 2주 차)** | 도메인 지식 부족 · 일감 우선순위 판단 어려움 · 읽기 위주 사용 | 1) 전체 일감 목록을 훑어 팀이 지금 무엇을 하는지 파악한다. 2) 자기에게 할당된 작은 일감을 열어 설명을 읽고 시작한다. 3) 막힐 때 일감 상태를 "진행 중"으로 두고 리드가 알아챌 수 있게 한다. |

## MVP 헌장과의 정합성

- **단일 테이블**: `tasks(id, title, description, assignee, status, created_at)` 하나로 세 페르소나의 9개 시나리오 모두 처리 가능.
- **기능 5개 이하**: ① 일감 생성 ② 목록 조회(담당자 필터 포함) ③ 담당자 변경 ④ 상태 토글(예: todo / doing / done) ⑤ 설명 편집.
- **1일 구현**: 위 5개 기능은 단일 라우트 그룹(`/tasks`)과 단일 API(`/api/tasks`)로 충분.
